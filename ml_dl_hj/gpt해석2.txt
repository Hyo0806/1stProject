원본: SUWON_S_DATA_TABLE_GENDER_SUM.csv

집계 단위: 일별

분석 대상

매출액: AMT_sum (일별 총매출)

날씨 변수

TEMP_mean : 일 평균 기온

RAIN_sum : 일 누적 강수량

2 상관관계 분석 결과
피어슨 상관계수
변수	AMT_sum과의 상관
기온(TEMP_mean)	+0.136
강수량(RAIN_sum)	−0.091

해석

기온 : 약한 양의 상관
→ 기온이 올라갈수록 매출이 조금 증가하는 경향

강수량 : 약한 음의 상관
→ 비가 올수록 매출이 감소하는 경향

하지만 둘 다 상관계수 절대값 < 0.2
날씨만으로 매출을 설명하기엔 영향이 매우 제한적

3. 날씨만 사용한 매출 예측 모델
모델 구성
   입력 변수 : 평균기온,  누적강수량
   모델 : RandomForestRegressor

데이터 분리
   Train 80% / Test 20%

예측 정확도 결과 ❌
   지표	값
   MAE	약 43,543,000원
   R² Score	-0.067
정확도 해석
   R² < 0  → 평균값으로 찍는 것보다도 못한 성능
즉, 날씨 정보만으로는 매출 예측이 불가능에 가깝다

왜 이런 결과가 나왔나? (중요 💡)
원인 분석

매출은 복합 결과

시간대(HOUR), 요일(DAY), 지역(DONG), 거래 건수(CNT)

 날씨는 “보조 변수” 역할, 단독 설명력 낮음
비가 와도 온라인 소비, 저녁 소비 등으로 상쇄 가능


핵심 결론
 ---> 날씨(기온·강수)는 매출에 ‘영향’은 있지만 ‘예측 변수’로는 단독 사용이 불가능하다.

반드시 포함해야 할 변수
HOUR (시간대)
DAY (요일)
DONG (지역)

날씨는 이렇게 써야 효과 있음
 - 서브 변수로 결합
 - 파생 변수화
 - 폭염 여부
 - 비 오는 날 여부
 - 체감온도

          +
       [날씨 보정]
