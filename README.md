## 🎯 해결하고자 한 문제

-   날씨가 실제 매출에 영향을 미치는가?
-   행정동·시간대별 매출 예측이 가능한가?
-   기상 데이터와 카드 소비 데이터를 결합하면 예측 정확도를 높일 수
    있는가?

------------------------------------------------------------------------

## 🏗 시스템 개요

본 프로젝트는 **공공 카드 소비 데이터 + 기상청 기후 데이터**를 결합하여\
행정동 단위 매출을 예측하는 ML 기반 분석 시스템입니다.

### 데이터 소스

  | 구분 | 출처 | 내용 |
  |------------------|------------------|-----------------------|
  |카드 소비 데이터|경기데이터드림|날짜·행정동·업종·매출|
  |행정동 코드|공공데이터포털|위치 매핑|
  |기상 데이터|기상자료개방포털|기온·강수량|

------------------------------------------------------------------------

## 🔄 전체 파이프라인 흐름

    데이터 수집
       ↓
    데이터 전처리 (이상치 제거, 집계, 병합)
       ↓
    상관관계 분석
       ↓
    ML/DL 모델 학습
       ↓
    성능 검증
       ↓
    Flask 웹 서비스 구현

------------------------------------------------------------------------

## 📊 데이터 전처리

### 1️⃣ 대용량 데이터 경량화

-   카드 소비 데이터 약 **3,226만 행**
-   음식 대분류 집계 후 한식 데이터 **44만 행으로 축소 (약 98% 감소)**
-   불필요 컬럼 제거 및 병합 로직 설계

### 2️⃣ 기상 데이터 정제

-   -50℃ 등 이상치 제거
-   시간대별 기온·강수 평균값 계산
-   행정동 기준 매출 데이터와 병합

------------------------------------------------------------------------

## 📈 상관관계 분석

-   기온 상승 시 매출 감소 경향
-   날씨와 매출 직접 상관관계는 낮음
-   이용 고객 수와 기온 간 일부 상관 확인

------------------------------------------------------------------------
